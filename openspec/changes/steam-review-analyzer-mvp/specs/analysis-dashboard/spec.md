## ADDED Requirements

### Requirement: 감성 분석 도넛 차트
시스템은 7개 카테고리별 감성 비율을 Recharts 도넛 차트로 시각화해야 한다(SHALL). 각 카테고리는 긍정(초록), 부정(빨강) 색상으로 구분된다.

#### Scenario: 차트 정상 렌더링
- **WHEN** 분석 결과 데이터가 대시보드에 전달되면
- **THEN** 각 카테고리에 대한 도넛 차트가 긍정/부정 비율로 렌더링된다

#### Scenario: 데이터 없는 카테고리
- **WHEN** 특정 카테고리의 total이 0인 경우
- **THEN** 해당 카테고리 차트는 "데이터 없음" 상태로 표시된다

### Requirement: 개선 우선순위 카드
시스템은 AI가 생성한 개선 우선순위를 순위별 카드 리스트로 표시해야 한다(SHALL). 각 카드는 순위, 카테고리, 핵심 문제, 대표 유저 발언을 포함한다.

#### Scenario: 우선순위 카드 정상 표시
- **WHEN** 피드백 데이터가 대시보드에 전달되면
- **THEN** 최대 5개의 우선순위 카드가 순위 순서로 표시된다

#### Scenario: 대표 유저 발언 인용
- **WHEN** 우선순위 카드가 렌더링되면
- **THEN** 각 카드는 관련 대표 유저 발언을 인용 형식으로 표시한다

### Requirement: 분석 로딩 상태
시스템은 리뷰 수집 및 AI 분석 중에 진행 상태를 사용자에게 시각적으로 표시해야 한다(SHALL).

#### Scenario: 분석 진행 중 로딩 표시
- **WHEN** 사용자가 분석 시작 버튼을 누르면
- **THEN** 시스템은 로딩 스피너와 현재 단계(리뷰 수집 중 / AI 분석 중) 메시지를 표시한다

#### Scenario: 분석 완료 후 결과 표시
- **WHEN** 분석이 완료되면
- **THEN** 로딩 상태가 사라지고 대시보드 결과가 즉시 렌더링된다

### Requirement: 결과 데이터 전달
시스템은 분석 결과를 sessionStorage에 저장하고, 대시보드 페이지는 이를 읽어 렌더링해야 한다(SHALL).

#### Scenario: 홈에서 대시보드로 이동
- **WHEN** 분석이 완료되면
- **THEN** 시스템은 결과를 sessionStorage에 저장하고 `/dashboard?appId=<appId>` 로 라우팅한다

#### Scenario: 직접 URL 접근 시 데이터 없음
- **WHEN** 사용자가 sessionStorage에 데이터 없이 대시보드 URL로 직접 접근하면
- **THEN** 시스템은 홈 페이지로 리다이렉트한다
