## ADDED Requirements

### Requirement: 빈 쿼리 검색 거부
시스템은 사용자가 빈 문자열로 검색을 시도할 때 검색을 실행하지 않아야 한다(SHALL).

#### Scenario: 빈 검색창에서 검색 버튼 클릭
- **WHEN** 사용자가 검색창을 비운 채 검색 버튼을 클릭하면
- **THEN** 검색 API 호출이 발생하지 않으며 오류 메시지 또는 비활성 상태로 처리된다

#### Scenario: 검색창에 공백만 입력 후 검색
- **WHEN** 사용자가 검색창에 공백(" ")만 입력하고 검색을 실행하면
- **THEN** 시스템은 400 오류를 반환하거나 검색을 실행하지 않는다

### Requirement: 검색 결과 없음 안내
시스템은 검색 결과가 0개일 때 사용자에게 안내 메시지를 표시해야 한다(SHALL).

#### Scenario: 존재하지 않는 게임명 검색
- **WHEN** 사용자가 "zzz_nonexistent_game_xyz_12345"처럼 결과가 없는 게임명을 검색하면
- **THEN** 시스템은 "검색 결과가 없습니다." 메시지를 화면에 표시한다

### Requirement: 대시보드 직접 접근 차단
시스템은 분석 데이터 없이 대시보드에 직접 접근하면 홈으로 리다이렉트해야 한다(SHALL).

#### Scenario: sessionStorage 없이 대시보드 URL 직접 접근
- **WHEN** 사용자가 분석을 거치지 않고 `/dashboard?appId=123` URL에 직접 접근하면
- **THEN** 시스템은 홈(`/`) 페이지로 자동 리다이렉트한다
